# 大数据圈人营销平台架构文档

## 1. 项目概述

基于现有Docker化大数据技术栈构建的智能圈人营销平台，提供从数据管理到人群匹配的全链路解决方案。

### 1.1 技术栈版本信息
- **Hadoop**: 3.3.6 (HDFS + YARN + MapReduce)
- **Spark**: 3.5.0 (大数据计算引擎)
- **Hive**: 4.0.1 (数据仓库)
- **HBase**: 2.5.6 (NoSQL实时存储)
- **Kafka**: latest (流数据)
- **Flink**: 1.17.1 (实时计算)
- **PostgreSQL**: latest (元数据存储)
- **Airflow**: latest (工作流调度)
- **ZooKeeper**: 3.8.3 (服务协调)

## 3. 四大平台详细设计

### 3.1 数据资产管理平台

**功能设计:**
- 标签元数据管理
- 数据质量检查
- 数据血缘管理  
- API接口提供

**技术实现:**
- **存储**: Hive数据仓库 + PostgreSQL元数据库
- **计算**: Spark进行数据质检
- **调度**: Airflow定时任务
- **接口**: RESTful API

**数据库设计:**
```sql
-- PostgreSQL元数据表
CREATE TABLE tag_metadata (
    tag_id VARCHAR(50) PRIMARY KEY,
    tag_name VARCHAR(100),
    tag_type VARCHAR(20),
    business_key VARCHAR(100),
    hive_table VARCHAR(100),
    hive_column VARCHAR(100),
    description TEXT,
    create_time TIMESTAMP,
    update_time TIMESTAMP
);

CREATE TABLE table_metadata (
    table_id VARCHAR(50) PRIMARY KEY,
    table_name VARCHAR(100),
    database_name VARCHAR(50),
    partition_column VARCHAR(50),
    description TEXT,
    create_time TIMESTAMP
);
```

**Hive表结构:**
```sql
-- 用户标签表 (按日分区)
CREATE TABLE user_tags (
    uid BIGINT,
    age_group STRING,
    gender STRING,  
    city STRING,
    consumption_level STRING,
    interest_tags ARRAY<STRING>
)
PARTITIONED BY (dt STRING)
STORED AS PARQUET;
```

### 3.2 圈人平台  

**功能设计:**
- 可视化规则配置
- 标签条件组合
- 定时调度计算
- 多渠道导出

**技术实现:**
- **规则引擎**: 基于JSON的规则配置
- **计算引擎**: Spark SQL
- **调度**: Airflow
- **存储**: HBase (实时查询) + Hive (离线数据)

**规则配置示例:**
```json
{
  "crowd_id": "crowd_001",
  "crowd_name": "高价值年轻用户",
  "rules": {
    "operator": "AND",
    "conditions": [
      {
        "tag": "age_group", 
        "operator": "in",
        "values": ["18-25", "26-35"]
      },
      {
        "tag": "consumption_level",
        "operator": ">=", 
        "values": ["高"]
      }
    ]
  },
  "schedule": {
    "cron": "0 2 * * *",
    "enabled": true
  },
  "export": {
    "channels": ["hbase", "hive", "file"]
  }
}
```

**Spark计算作业:**
```sql
-- 转换为Spark SQL
SELECT uid, '{crowd_id}' as crowd_code, dt
FROM user_tags 
WHERE dt = '${yesterday}'
  AND age_group IN ('18-25', '26-35')  
  AND consumption_level >= '高'
```

### 3.3 活动平台

**功能设计:**
- 活动创建与管理
- 人群绑定
- 效果统计
- 时间窗口控制

**技术实现:**
- **存储**: PostgreSQL (活动配置) + HBase (结果数据)
- **查询**: 联合查询HBase和PostgreSQL

**数据库设计:**
```sql
-- 活动配置表
CREATE TABLE campaigns (
    campaign_id VARCHAR(50) PRIMARY KEY,
    campaign_name VARCHAR(100),
    crowd_id VARCHAR(50),
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    status VARCHAR(20),
    description TEXT,
    create_time TIMESTAMP
);

-- 活动统计表  
CREATE TABLE campaign_stats (
    campaign_id VARCHAR(50),
    stat_date DATE,
    target_users BIGINT,
    reached_users BIGINT,
    conversion_users BIGINT,
    PRIMARY KEY (campaign_id, stat_date)
);
```

### 3.4 人群匹配平台

**功能设计:**
- 实时uid查询
- 人群归属判断
- 高并发支持
- 毫秒级响应

**技术实现:**
- **存储**: HBase
- **缓存**: 可扩展Redis
- **接口**: RESTful API

**HBase表设计:**
```

  - cf:crowd_001 = "1" (属于该人群)
  - cf:crowd_002 = "1"

```

## 5. 数据流设计

### 5.1 离线数据处理流程

```
数据源 → Kafka → HDFS → Hive → Spark计算 → HBase/Hive输出
   ↓
标签数据ETL ← Airflow调度 ← 质量检查 ← 数据清洗
```

### 5.2 实时数据处理流程  

```
实时数据 → Kafka → Flink → HBase → API查询
                    ↓
                 实时标签更新
```

### 5.3 人群计算流程

```
规则配置 → Spark SQL生成 → Hive数据查询 → 结果写入HBase → API提供查询
   ↓
Airflow调度 → 定时更新 → 增量计算 → 结果合并
```

## 6. 部署与初始化

### 6.1 环境启动

```bash
# 1. 构建所有镜像
./rebuild.sh

# 2. 启动所有服务
docker compose up -d

# 3. 初始化HDFS
chmod +x init-hdfs.sh
./init-hdfs.sh

# 4. 启动历史服务器
docker compose up -d historyserver

# 5. 检查服务状态
docker compose ps
```

### 6.2 数据库初始化

**Hive数据库创建:**
```sql
-- 连接Hive
docker exec -it hive-server beeline -u "jdbc:hive2://hive-server:10000"

-- 创建数据库
CREATE DATABASE marketing_platform;
USE marketing_platform;

-- 创建用户标签表
CREATE TABLE user_tags (
    uid BIGINT,
    age_group STRING,
    gender STRING,  
    city STRING,
    consumption_level STRING,
    interest_tags ARRAY<STRING>,
    update_time STRING
)
PARTITIONED BY (dt STRING)
STORED AS PARQUET
LOCATION '/user/hive/warehouse/marketing_platform.db/user_tags';
```

**HBase表创建:**
```bash
# 进入HBase Shell  
docker exec -it hbase-master hbase shell

# 创建人群映射表
create 'user_crowd_mapping', 'cf'

# 创建人群元数据表
create 'crowd_metadata', 'info', 'stats'
```

**PostgreSQL表创建:**
```sql
-- 连接PostgreSQL
docker exec -it postgres-metastore psql -U hive -d metastore_db

-- 创建业务表
\i /path/to/init_marketing_tables.sql
```

### 6.3 Airflow配置

```bash
# 1. 生成Fernet密钥
docker compose run --rm airflow-webserver python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# 2. 更新.env文件添加密钥
echo "AIRFLOW__CORE__FERNET_KEY=<生成的密钥>" >> .env

# 3. 初始化数据库
docker compose run --rm airflow-webserver airflow db init

# 4. 创建管理员用户
docker compose run --rm airflow-webserver airflow users create \
  --username admin --firstname Marketing --lastname Admin --role Admin \
  --email admin@marketing.com --password admin123

# 5. 重启服务
docker compose restart airflow-webserver airflow-scheduler
```

## 7. API接口设计

### 7.1 数据资产管理API

**标签元数据管理:**
```http
GET    /api/v1/tags                    # 获取标签列表
POST   /api/v1/tags                    # 创建标签
PUT    /api/v1/tags/{tag_id}          # 更新标签
DELETE /api/v1/tags/{tag_id}          # 删除标签

GET    /api/v1/tables                  # 获取表列表  
POST   /api/v1/tables/{table}/data     # 写入数据
```

### 7.2 圈人平台API

**人群管理:**
```http
GET    /api/v1/crowds                  # 获取人群列表
POST   /api/v1/crowds                  # 创建人群
PUT    /api/v1/crowds/{crowd_id}       # 更新人群规则
DELETE /api/v1/crowds/{crowd_id}       # 删除人群

POST   /api/v1/crowds/{crowd_id}/compute    # 触发计算
GET    /api/v1/crowds/{crowd_id}/status     # 查看计算状态
GET    /api/v1/crowds/{crowd_id}/export     # 导出人群数据
```

### 7.3 人群匹配API

**实时查询:**
```http
GET    /api/v1/users/{uid}/crowds      # 查询用户所属人群
POST   /api/v1/users/batch-query       # 批量查询用户人群
GET    /api/v1/crowds/{crowd_id}/users # 查询人群用户列表
```

## 8. 监控与运维

### 8.1 服务监控

**健康检查脚本:**
```bash
#!/bin/bash
# health-check.sh

echo "=== 服务健康检查 ==="
services=("namenode:9870" "resourcemanager:8088" "hbase-master:16010" 
         "hive-server:10002" "flink-jobmanager:8081" "airflow-webserver:8080")

for service in "${services[@]}"; do
    if curl -f -s "http://localhost:${service##*:}" > /dev/null; then
        echo "✅ ${service%:*} - 正常"
    else  
        echo "❌ ${service%:*} - 异常"
    fi
done
```
